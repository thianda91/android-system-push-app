# 安卓混合推送集成功能清单

## 一、项目概述

### 1.1 项目目标
- 实现安卓应用在无后台驻留情况下接收推送消息
- 采用混合推送策略，确保各品牌设备上的高送达率
- 支持用户订阅角色，实现定向推送

### 1.2 技术架构
- 客户端：集成FCM和主要厂商推送SDK
- 服务端：统一推送管理，支持多通道智能路由
- 数据层：维护设备token映射关系，支持用户角色管理

## 二、混合推送架构设计

### 2.1 整体架构
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 应用服务器  │────>│ 推送管理服务 │────>│ 各推送通道  │
└─────────────┘     └─────────────┘     └─────────────┘
                                      ┌─────────────┐
                                      │  FCM        │
                                      ├─────────────┤
                                      │  HMS Push   │
                                      ├─────────────┤
                                      │  MI Push    │
                                      ├─────────────┤
                                      │  OPPO Push  │
                                      ├─────────────┤
                                      │  vivo Push  │
                                      ├─────────────┤
                                      │  Meizu Push │
                                      └─────────────┘
```

### 2.2 核心流程
1. 应用启动时，同时初始化多个推送SDK
2. 获取各通道token并上传到服务器
3. 服务器根据设备类型和推送优先级选择最优通道
4. 消息通过选定通道推送到设备
5. 客户端统一处理收到的消息

## 三、推送通道集成

### 3.1 Google FCM集成
- **依赖配置**：`com.google.firebase:firebase-messaging:23.2.1`
- **核心功能**：
  - 初始化Firebase服务
  - 实现`FirebaseMessagingService`处理消息
  - 获取FCM token并上传
  - 处理token刷新

### 3.2 华为HMS Push Kit集成
- **依赖配置**：`com.huawei.hms:push:6.11.0.300`
- **核心功能**：
  - 初始化HMS Core
  - 获取华为推送token
  - 实现`HmsMessageService`处理消息
  - 处理透传消息和通知消息

### 3.3 小米MI Push集成
- **依赖配置**：AAR包集成
- **核心功能**：
  - 初始化小米推送SDK
  - 自定义`PushMessageReceiver`
  - 获取RegID并上传
  - 处理通知点击事件

### 3.4 OPPO Push集成
- **依赖配置**：`com.heytap.msp`
- **核心功能**：
  - 初始化OPPO推送服务
  - 注册推送服务
  - 处理消息回调
  - 配置通知渠道

### 3.5 vivo Push集成
- **依赖配置**：`vivo:vpush:4.1.0.0`
- **核心功能**：
  - 初始化vivo推送服务
  - 获取regId并上传
  - 实现`OpenClientPushMessageReceiver`
  - 处理通知点击事件

### 3.6 魅族Meizu Push集成
- **依赖配置**：`com.meizu.flyme.internet:push-internal:5.0.3`
- **核心功能**：
  - 初始化魅族推送SDK
  - 注册消息接收广播
  - 实现`MzPushMessageReceiver`
  - 处理标签和别名

### 3.7 三星Samsung Push集成
- **依赖配置**：Samsung Cloud Push SDK
- **核心功能**：
  - 初始化三星推送服务
  - 获取registrationId
  - 处理推送消息
