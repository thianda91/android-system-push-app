# 安卓推送服务端设计与通道配置指南

## 一、服务端架构设计

### 1.1 系统架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 客户端应用    │────>│ 推送服务端    │────>│ 各厂商推送平台 │
└──────────────┘     └──────────────┘     └──────────────┘
                      ┌──────────────┐     ┌──────────────┐
                      │ 数据库       │<────│ 监控统计模块  │
                      └──────────────┘     └──────────────┘
```

### 1.2 核心模块

#### 1.2.1 客户端接口模块
- **功能**：处理客户端请求，包括注册、获取token、配置角色等
- **API接口**：
  - `/api/push/register` - 注册设备
  - `/api/push/token` - 上传推送token
  - `/api/push/role` - 配置用户角色
  - `/api/push/unregister` - 注销设备

#### 1.2.2 推送管理模块
- **功能**：管理推送任务，选择最优推送通道
- **核心功能**：
  - 多通道消息路由
  - 推送任务调度
  - 消息队列管理
  - 失败重试机制

#### 1.2.3 厂商通道集成模块
- **功能**：对接各厂商推送平台
- **支持通道**：
  - FCM (Google)
  - HMS Push (华为)
  - MI Push (小米)
  - OPPO Push (OPPO)
  - vivo Push (vivo)
  - Meizu Push (魅族)
  - Samsung Push (三星)

#### 1.2.4 用户管理模块
- **功能**：管理用户信息和角色
- **核心功能**：
  - 用户角色管理
  - 推送权限控制
  - 设备管理
  - 用户分组

#### 1.2.5 监控统计模块
- **功能**：监控推送效果，统计数据
- **核心功能**：
  - 推送到达率统计
  - 延迟时间监控
  - 失败原因分析
  - 数据报表生成

### 1.3 数据模型

## 二、各推送通道配置步骤

### 2.1 Google FCM 配置步骤

#### 2.1.1 开发者操作步骤
1. **注册Google开发者账号**
   - 访问 [Google Cloud Console](https://console.cloud.google.com/)
   - 注册并登录账号

2. **创建项目**
   - 点击"创建项目"
   - 输入项目名称
   - 选择组织（可选）
   - 点击"创建"

3. **启用FCM服务**
   - 在项目中，导航到"API和服务" > "仪表板"
   - 点击"启用API和服务"
   - 搜索"Firebase Cloud Messaging"
   - 点击"启用"

4. **创建应用**
   - 访问 [Firebase控制台](https://console.firebase.google.com/)
   - 选择或创建项目
   - 点击"添加应用"
   - 选择"Android"
   - 输入应用包名
   - 点击"注册应用"

5. **获取配置文件**
   - 下载`google-services.json`文件
   - 将文件添加到Android项目的app模块根目录

6. **获取服务端密钥**
   - 在Firebase控制台中，导航到"项目设置" > "云消息传递"
   - 复制"服务器密钥"
   - 保存用于服务端调用

#### 2.1.2 服务端配置
- **依赖**：使用Firebase Admin SDK或直接调用FCM REST API
- **认证**：使用服务端密钥
- **API端点**：`https://fcm.googleapis.com/v1/projects/{project-id}/messages:send`
- **请求格式**：JSON格式，包含target token和消息内容

### 2.2 华为HMS Push Kit 配置步骤

#### 2.2.1 开发者操作步骤
1. **注册华为开发者账号**
   - 访问 [华为开发者联盟](https://developer.huawei.com/consumer/cn/)
   - 注册并完成实名认证

2. **创建项目**
   - 登录 [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)
   - 点击"创建项目"
   - 输入项目名称
   - 点击"确定"

3. **添加应用**
   - 在项目中，点击"添加应用"
   - 选择"Android"
   - 输入应用包名
   - 选择应用类别
   - 点击"确定"

4. **启用Push Kit**
   - 在应用中，导航到"增长" > "Push Kit"
   - 点击"启用服务"

5. **获取配置文件**
   - 点击"下载agconnect-services.json"
   - 将文件添加到Android项目的app模块根目录

6. **获取服务端凭证**
   - 在AppGallery Connect中，导航到"项目设置" > "API管理"
   - 创建OAuth 2.0客户端
   - 获取Client ID和Client Secret

#### 2.2.2 服务端配置
- **依赖**：使用华为Push Kit服务端SDK或调用REST API
- **认证**：使用OAuth 2.0获取access token
- **API端点**：`https://push-api.cloud.huawei.com/v1/messages:send`
- **请求格式**：JSON格式，包含target token和消息内容

## 三、服务端API设计

### 3.1 客户端API

#### 3.1.1 设备注册
- **接口**：`POST /api/push/register`
- **请求参数**：
  ```json
  {
    "device_id": "设备唯一标识",
    "device_type": "Android",
    "brand": "华为",
    "model": "Mate 40 Pro",
    "os_version": "10.0",
    "app_version": "1.0.0",
    "nickname": "用户昵称"
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "注册成功",
    "data": {
      "device_id": "设备唯一标识",
      "status": "registered",
      "last_active_at": "2026-04-13T12:00:00"
    }
  }
  ```

#### 3.1.2 上传推送Token
- **接口**：`POST /api/push/token`
- **请求参数**：
  ```json
  {
    "device_id": "设备唯一标识",
    "tokens": [
      {
        "channel_type": "FCM",
        "token": "FCM token",
        "expires_at": "2026-12-31T23:59:59"
      },
      {
        "channel_type": "HMS",
        "token": "HMS token",
        "expires_at": "2026-12-31T23:59:59"
      }
    ]
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "Token上传成功",
    "data": {
      "device_id": "设备唯一标识",
      "tokens": ["FCM", "HMS"]
    }
  }
  ```

#### 3.1.3 配置用户角色
- **接口**：`POST /api/push/role`
- **请求参数**：
  ```json
  {
    "device_id": "设备唯一标识",
    "roles": ["premium", "user"]
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "角色配置成功",
    "data": {
      "device_id": "设备唯一标识",
      "roles": ["premium", "user"]
    }
  }
  ```

#### 3.1.4 设置昵称
- **接口**：`POST /api/push/nickname`
- **请求参数**：
  ```json
  {
    "device_id": "设备唯一标识",
    "nickname": "新昵称"
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "昵称设置成功",
    "data": {
      "device_id": "设备唯一标识",
      "nickname": "新昵称",
      "last_active_at": "2026-04-13T12:00:00"
    }
  }
  ```

#### 3.1.5 设备注销
- **接口**：`POST /api/push/unregister`
- **请求参数**：
  ```json
  {
    "device_id": "设备唯一标识"
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,

    "data": {
      "device_id": "设备唯一标识",
  ```

### 3.2 管理API

#### 3.2.1 发送推送
- **接口**：`POST /api/admin/push/send`
- **请求参数**：
  ```json
  {
    "target_type": "device", // device, role, all
    "target_id": "设备唯一标识",
    "title": "消息标题",
    "content": "消息内容",
    "data": {
      "key1": "value1",
      "key2": "value2"
    },
    "priority": "high",
    "expire_time": 3600
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "推送任务创建成功",
    "data": {
      "task_id": "任务ID",
      "status": "pending"
  ```

## 四、服务端部署与运维

### 4.1 技术选型

#### 4.1.1 后端框架
- **可选框架**：Spring Boot, Node.js Express, Django等
- **推荐**：Spring Boot (Java)
  - 成熟稳定
  - 生态丰富
  - 性能优异

#### 4.1.2 数据库
- **可选**：MySQL, PostgreSQL, MongoDB
- **推荐**：MySQL
  - 关系型数据库，适合结构化数据
  - 社区活跃，支持完善

#### 4.1.3 消息队列
- **可选**：RabbitMQ, Kafka, Redis
- **推荐**：RabbitMQ
  - 可靠性高
  - 支持多种消息模式
  - 易于集成

#### 4.1.4 缓存
- **可选**：Redis, Memcached
- **推荐**：Redis
  - 性能优异
  - 支持多种数据结构
  - 可用于token管理和速率限制

### 4.2 部署方案

#### 4.2.1 容器化部署
- 使用Docker容器化
- 配置Docker Compose或Kubernetes
- 实现自动扩缩容

#### 4.2.2 环境配置
- 开发环境：本地或开发服务器
- 测试环境：独立测试服务器
- 生产环境：高可用集群
